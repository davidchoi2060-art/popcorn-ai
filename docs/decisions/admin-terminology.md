# 관리자 용어집 (운영자 대면 명명 규약) — 확정 2026-07-07

> **단일 원천.** 관리자(운영자) 화면의 모든 라벨·용어는 이 표를 따른다.
> 새 화면·컴포넌트를 만들 때 여기 없는 은어를 만들지 말고, 먼저 이 표에 등록한 뒤 사용한다.

## 명명 원칙
1. 대상 사용자는 개발자가 아니라 **용산 매장 운영자**다. 이들의 언어(입고·검수·재고·매입·주문·손님)로 맞춘다.
2. **명사 나열이 아니라 "무엇을 하는 곳"**이 드러나게 짓는다.
3. **영어 기술 은어 퇴출**: 큐(queue)·풀(pool)·import·verified·candidate·log·relay·session 등. 단 부품 규격(CPU/GPU/DDR5/RTX 등)은 그대로 둔다.

## 핵심 개념 (원자 — 여기서 모든 화면명이 파생)
| 개념(뜻) | 확정 용어 | 폐기된 은어 |
|---|---|---|
| AI가 확신 못한 상품 정보를 사람이 최종 확인 | **상품 검수** (목록 = 검수 대기) | 검수 큐 |
| 업로드해서 얼려둔 원본 데이터 | **상품 원본 자료** | 원본 조회 / imports |
| 검수·재고를 통과해 추천에 쓸 수 있는 부품 집합 | **추천 가능 재고** | 후보 풀 / candidate pool |
| 호환 판정에 꼭 필요한 사양 / 그 확인 여부 | **필수 사양 / 확인 완료** | 치명필드 / verified |
| 확정한 필드를 재적재가 못 덮어쓰게 고정 | **확정(고정)** | locked_fields 잠금 |

## 검수 사유 배지 (review_type)
| DB 값 | 화면 라벨 |
|---|---|
| spec_conflict | **사양 불일치** |
| spec_missing | **필수 사양 없음** |
| low_confidence | **AI 확신 낮음** |
| csv_error | **등록 오류** |
| sourcing_hold | **매입 대기** |

## 기타 개념어
| 은어 | 확정 |
|---|---|
| 매입(sourcing) | 용산 도매서 사오는 것 = **매입** |
| 교차검증(원문 vs AI 지식 대조) | **사양 대조** |
| 릴레이(relay) | **쇼핑몰 전달** |
| 스왑(swap) | **부품 교체** |
| 세션(session) | **상담** |
| verified 커버리지 | **확인 완료율** |
