> **단일 원천.** 모든 관리자(ADM-\*) 화면의 존재 근거·범위·우선순위는 이 문서를 따른다.
> 용어는 [`admin-terminology.md`](admin-terminology.md), 데이터는 [`../06_db-erd.md`](../06_db-erd.md)(ERD v3)를 표준으로 한다.

## 정체성 (한 줄)
고객 화면이 "왜 이 견적인가"를 말한다면, **관리자는 그 이유가 사실이 되도록 뒤에서 보증하는 곳** —
즉 **엔진이 거짓말하지 않도록 재고·사양·규칙을 신뢰 가능한 상태로 유지하는 곳.**

## 4대 원칙 (화면 취사선택의 잣대)
1. **쇼핑몰 백오피스가 아니다.** 주문·결제·회원 원장은 기존 팝콘PC 쇼핑몰. 이 관리자의 고유 임무는 **AI 견적 엔진의 데이터 품질**뿐.
2. **모든 작업의 목적은 하나** — "확인 완료된 필수 사양"을 늘려 **추천 가능 재고를 키우는 것**.
3. **운영자는 판단자다, 입력자가 아니다.** AI가 추출, 사람은 AI가 못 정한 것만 판정하고 규칙을 정한다. 화면은 타이핑 최소·결정 최대.
4. **재현성의 수호.** 결정론 엔진이라 운영자의 확정·규칙·가중치 하나가 미래 추천을 바꾼다. → 확정(고정)·작업 기록(감사)이 필수 부속.

## 확정된 경계 결정
- **주문·고객 = 읽기 전용 창.** 여기서 주문을 *생성하지 않는다*(원칙 1). 쇼핑몰로 딥링크하고, 관리자는 상태·이력만 조회. (기존 미정 "주문 경계" 해소)

## 확정된 화면 정의 (점검 통과분 누적)

### 상품 검수 (ADM-PRD-020) — 점검 완료 2026-07-07
- **목적:** AI가 자동 통과 못 시킨 상품 정보를 운영자가 판정해 "확인 완료된 필수 사양"으로 만들어 추천 가능 재고에 올린다.
- **핵심 동작:** 위험도순으로 하나씩 → 사양 대조·필수 사양 확인 → 확정(원문/지식/직접수정) 또는 반려. 타이핑 최소·결정 최대.
- **핵심 데이터/상태:** review_type(4: 사양 불일치/필수 사양 없음/AI 확신 낮음/등록 오류) · confidence · 필수 사양 체크리스트 · locked_fields.
- **주요 컴포넌트:** 마스터-디테일 · 사유 필터 · 위험도 정렬(치명>주의>경미) · 사양 대조 카드(+원문 딥링크) · 4단계 파이프라인 · 필수 사양 체크리스트 · 액션.
- **예외:** 빈 큐 / 저신뢰지만 값 정상 / 등록 오류(사양 판정 이전 단계).
- **경계:** 검수는 **사양만** 판정. 재고·매입(구 sourcing_hold)은 여기서 다루지 않고 **매입 견적**으로 이관.
- **적용된 개선:** #1 매입 대기 제거 ✅ · #2 원문 딥링크 ✅ · #4 위험도 정렬/표시 ✅.
- **보류(코어 스파인 완성 후 일괄):** #3 확정 결과 피드백("추천 가능 재고 +1·영향 견적 N") · #5 사유별 일괄 확정 · #6 되돌리기·작업 기록 링크.

### 상품 일괄 등록 (ADM-CSV-010) — 점검 완료 2026-07-13
- **목적:** 엑셀/CSV를 원본 자료(동결)에 적재하고, 적재가 스파인 전체에 일으킬 일을 미리 보여준 뒤 실행하게 한다.
- **핵심 동작:** 3단계 마법사 — ①파일·매핑(프리셋 자동 매칭, 예외만 확인) → ②검증 미리보기(영향 예측) → ③적재 실행·배치 진행.
- **핵심 데이터/상태:** 배치 ID · 행 분류(신규/갱신/공급처 가격 추가/오류) · 매핑 프리셋 · 잠금 충돌 · SKU.
- **주요 컴포넌트:** 드롭존 · 공급처 프리셋 · 매핑 테이블 · 영향 예측 카드 7종 · 오류 행/잠금 충돌 미리보기 · 배치 4단계 진행 · 결과 요약+후속 CTA.
- **예외:** 오류 행(제외·원본 보존) · SKU 없는 행(공급처+모델명 후보 매칭→검수 확정) · 잠금 충돌(잠금 우선 무시).
- **확정된 결정:** 중복 판정 키 = **자체 상품코드(SKU)**. 단 **같은 SKU의 매입가는 공급처별로 별도 관리**(상품 1 : N 공급처 가격) — 미리보기에 "공급처 가격 추가" 분류가 존재하는 이유.
- **경계:** 원본은 이 화면에서 수정 불가(동결). 교정 = 재업로드 또는 검수 확정. 배치 취소는 일괄 등록 이력(P3) 소관.

### 상품 목록 (ADM-PRD-010) — 점검 완료 2026-07-13
- **목적:** 정본(단일 진실)을 열람하고, 각 상품이 **왜 추천 가능/불가인지**를 상태와 함께 즉시 보이게 한다.
- **핵심 동작:** 상태 필터(추천 가능/검수 대기/재고 없음/가격 검토) → 불가 사유 배지에서 해당 처리 화면으로 바로가기.
- **핵심 데이터/상태:** SKU · 필수 사양 확인(n/4) · 재고 · 공급처 수 · 판매가 · 추천 가능 상태+사유.
- **주요 컴포넌트:** KPI 스트립 4종 · 상태 필터 pills · 검색 · 테이블(사유별 액션 버튼) · 단건 등록 진입.
- **예외:** 재고 0(빨강 강조) · 조건 무결과.

### 상품 상세·등록·수정 (ADM-PRD-030/040) — 점검 완료 2026-07-13
- **목적:** 필드 단위로 **값의 출처(원본 추출/AI 대조/운영자 확정)와 잠금 상태**를 보며 편집한다.
- **핵심 동작:** 필수 사양 필드별 확인·잠그기(잠그면 출처=운영자 확정, 편집 불가) · 직접 수정=확정→잠금 · 공급처별 매입가 관리(최저가 표시).
- **핵심 데이터/상태:** 필드별 출처·잠금 · 공급처 1:N 매입가 · 상품 흐름(원본 반입/검수 이력/추천 가능 여부).
- **주요 컴포넌트:** 필수 사양 패널(출처 배지+잠금 토글) · 기본 정보 폼 · 공급처별 매입가 표 · 흐름 카드(원본/검수/추천 딥링크).
- **예외:** 미확인 필드(검수 대기 배지→검수 딥링크) · ?new=1 등록 모드(SKU 자동 발번).

### 상품 원본 자료 (ADM-IMP-010) — 점검 완료 2026-07-13
- **목적:** 동결 원본을 받은 그대로 열람(읽기 전용) — 검수 "원문 상세 보기"의 종착지.
- **핵심 동작:** 배치 목록 → 행 드릴다운. `?id=SKU` 딥링크 시 해당 배치 자동 선택 + 행 강조.
- **핵심 데이터/상태:** 배치(파일명·행수·상태) · 행(원본 셀 그대로·처리 결과: 정본 반영/검수 회부/오류 제외/잠금 우선).
- **주요 컴포넌트:** 배치 마스터 리스트 · 행 테이블(monospace 원본) · 처리 결과 배지(관련 화면 딥링크) · 읽기 전용 배지.
- **예외:** 오류 행(적재 제외·원본 보존) · 잠금 우선(확정값 유지 표시).

**→ 상품 그룹 5화면 전부 점검 완료** (검수·일괄 등록·목록·상세/등록·원본 자료).

### 가격·매입 4화면 — 2026-07-13 일괄 생성
- **가격 검토 대기(ADM-PRC-010)**: 임계(±15%)·마진 위반 건만 수집. 제안 판매가=매입가+마진 정책 자동 적용. 판정(승인/유지/수정)→가격 이력·작업 기록.
- **마진 정책(ADM-PRC-020)**: 카테고리별 기본/최소 마진·끝자리 규칙. **버전 발행(v8→v9)** — 소급 없음, 과거 견적은 당시 버전으로 재현.
- **매입 견적(ADM-SRC-010)**: 재고 소진·안전재고 미달 자동 등록 → 견적 요청 → 공급처 회신 비교(최저 표시) → **확정 시 추천 가능 재고 재편입**. (검수에서 분리한 매입 대기의 새 집)
- **가격 이력(ADM-PRC-030)**: 판매가+공급처별 매입가 추이(SVG) + 변동 기록(사유가 배치/가격 검토/운영자로 링크 — 감사 연결).

### 추천 설정 3화면 — 2026-07-13 일괄 생성
- **조립 호환 규칙·필수 사양(ADM-ENG-010)**: 규칙 R-01~06 토글+파라미터(여유율%), 카테고리별 필수 사양 필드 정의(검수 crit와 동일 소스). **규칙 버전 발행 시 추천 가능 재고 재계산** 경고.
- **추천 기준 설정(ADM-ENG-020)**: 가중치 5축 슬라이더(합계 100 강제), 프리셋(가성비/균형/고성능), 변경 미리보기(동일 제약 견적 변화), **정책 버전 발행**.
- **추천 가능 재고 현황(ADM-ENG-030)**: S1 카운터의 뒷단. 카테고리별 스택 막대(가능/검수/재고/가격), 제외 사유 합계(각 처리 화면 링크), **얇은 카테고리 경고**(견적 실패 위험→매입 연결).

### 견적·주문·고객 4화면 — 2026-07-13 일괄 생성 (읽기 전용 계층)
- **견적 상담 기록(ADM-ORD-010)**: 세션 목록+상세(제약객체 칩·결과 구성·**스냅샷 ID·"이 스냅샷으로 재현"**). 재현=소명 근거.
- **주문(ADM-ORD-020)**: 원장=쇼핑몰 고지. 전달 상태(전달/재시도/실패)만 관리, 실패 시 재전달. 쇼핑몰 딥링크.
- **부품 교체·클릭 기록(ADM-ORD-030)**: 상위 교체 패턴+신호(추천 기준/규칙/매입으로 링크), 근거 리포트 클릭률(신뢰 포지셔닝 지표).
- **고객(ADM-CUS-010)**: 마스킹·최소 정보(상담·주문 수·최근 활동)만. 상세=쇼핑몰.

### 시스템 4화면 — 2026-07-13 일괄 생성
- **AI 사용량·비용(ADM-SYS-010)**: 일 캡($50) 게이지, 용도별(추출 72%/파싱 21%/근거 7%), 배치당 비용, **캡 도달 정책**(고객 대면 유지·배치 연기).
- **운영자·권한(ADM-SYS-020)**: 역할 3종(조회/운영자/관리자) + 허용 범위 매트릭스(정책 발행은 관리자 전용).
- **작업 기록(ADM-SYS-030)**: 감사 로그(확정·잠금/정책·규칙 발행/가격/적재/매입) 필터. 타 화면 데이터와 정합.
- **일괄 등록 이력(ADM-SYS-040)**: 배치 결과+**배치 취소(72h 내, 정본 롤백·원본 보존)** — 일괄 등록 검토 #6의 집.
