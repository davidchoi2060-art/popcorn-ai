## 정체성 (한 줄, A-10 개정)
고객 화면이 "왜 이 견적인가"를 말한다면, **관리자는 그 이유가 사실이 되도록 뒤에서 보증하고, 견적이 판매로 완결되도록 책임지는 곳** —
엔진이 거짓말하지 않도록 재고·사양·규칙을 유지하고, **주문·결제·배송·환불까지 본 시스템의 원장으로 처리**한다.

## 4대 원칙 (화면 취사선택의 잣대)
1. ~~쇼핑몰 백오피스가 아니다~~ → **A-10 개정: 본 시스템이 커머스 원장이다.** 회원·주문·결제·배송·환불을 직접 처리하되, 기존 쇼핑몰은 전환기 **병행 채널** — 도메인별 **운영 모드 스위치**(회원 연동·결제·정산·배송·환불, 각 `자체`/`쇼핑몰`)로 분기한다. 어떤 모드든 **주문 원장은 항상 본 시스템에 생성**(인계=복제 전달).
2. **모든 작업의 목적은 하나** — "확인 완료된 필수 사양"을 늘려 **추천 가능 재고를 키우는 것**. (+A-10: **재고 차감 단일 원장 = 본 시스템.** 쇼핑몰 판매분은 API 유입으로 정합, 결제 진행 중 재고는 예약(hold).)
3. **운영자는 판단자다, 입력자가 아니다.** AI가 추출, 사람은 AI가 못 정한 것만 판정하고 규칙을 정한다. 화면은 타이핑 최소·결정 최대.
4. **재현성의 수호.** 결정론 엔진이라 운영자의 확정·규칙·가중치 하나가 미래 추천을 바꾼다. → 확정(고정)·작업 기록(감사)이 필수 부속. (+A-10: **운영 모드 스위치도 설정 버전 + 작업 기록 + 관리자 전용.**)

### 상품 검수 (ADM-PRD-020) — 점검 완료 2026-07-07

- **핵심 동작:** 위험도순으로 하나씩 → 사양 대조·필수 사양 확인 → 확정(원문/지식/직접수정) 또는 반려. 타이핑 최소·결정 최대.
- **핵심 데이터/상태:** review_type(4: 사양 불일치/필수 사양 없음/AI 확신 낮음/등록 오류) · confidence · 필수 사양 체크리스트 · locked_fields.
- **주요 컴포넌트:** 마스터-디테일 · 사유 필터 · 위험도 정렬(치명>주의>경미) · 사양 대조 카드(+원문 딥링크) · 4단계 파이프라인 · 필수 사양 체크리스트 · 액션.
- **예외:** 빈 큐 / 저신뢰지만 값 정상 / 등록 오류(사양 판정 이전 단계).
- **경계:** 검수는 **사양만** 판정. 재고·매입(구 sourcing_hold)은 여기서 다루지 않고 **매입 견적**으로 이관.

### 상품 일괄 등록 (ADM-CSV-010) — 점검 완료 2026-07-13
- **목적:** 엑셀/CSV를 원본 자료(동결)에 적재하고, 적재가 스파인 전체에 일으킬 일을 미리 보여준 뒤 실행하게 한다.
- **핵심 동작:** 3단계 마법사 — ①파일·매핑(프리셋 자동 매칭, 예외만 확인) → ②검증 미리보기(영향 예측) → ③적재 실행·배치 진행.
- **핵심 데이터/상태:** 배치 ID · 행 분류(신규/갱신/공급처 가격 추가/오류) · 매핑 프리셋 · 잠금 충돌 · SKU.
- **주요 컴포넌트:** 드롭존 · 공급처 프리셋 · 매핑 테이블 · 영향 예측 카드 7종 · 오류 행/잠금 충돌 미리보기 · 배치 4단계 진행 · 결과 요약+후속 CTA.
- **예외:** 오류 행(제외·원본 보존) · SKU 없는 행(공급처+모델명 후보 매칭→검수 확정) · 잠금 충돌(잠금 우선 무시).
- **확정된 결정:** 중복 판정 키 = **자체 상품코드(SKU)**. 단 **같은 SKU의 매입가는 공급처별로 별도 관리**(상품 1 : N 공급처 가격) — 미리보기에 "공급처 가격 추가" 분류가 존재하는 이유.
- **경계:** 원본은 이 화면에서 수정 불가(동결). 교정 = 재업로드 또는 검수 확정. 배치 취소는 일괄 등록 이력(P3) 소관.

### 단가표 일일 반영 (ADM-PRC-040) — 신설 2026-07-21
- **목적:** 거래처가 1일 1회 보내는 단가표 엑셀을 어제와 비교해 **변경분만** 확인·반영한다(무변동 행 자동 통과 — 매일 아침 3분 루틴).
- **핵심 동작:** 파일 업로드 → 공급처 프리셋 자동 인식(다중 시트·상이 레이아웃·헤더 2줄·칩셋 그룹 캐리포워드 정규화, 규정/빈 시트 제외) → diff 3분류 확인 → 선택 일괄 반영.
- **핵심 데이터/상태:** 공급처 프리셋 · 모델명↔SKU 매핑 테이블(1회 확정 후 기억) · diff 분류(가격 변동/신규 모델/발주 상태 전환 품절·가능·문의) · 판매가 산정식(매입가+카드수수료 2.2%+마진 0%, 마진 정책 연동).
- **주요 컴포넌트:** 파일·프리셋 인식 카드 · 판매가 공식 스트립 · KPI 4종 · diff 탭 3종 · 선택 반영(되돌리기+가격 이력 링크).
- **예외:** 미매핑 신규 모델(후보 매칭→검수 확정) · 가격지도(단가 단속) 시트는 **미적용 명시**(단품 외부 노출 없음 — 2026-07-21 사용자 확인).
- **근거:** 실제 MSI_단가표0716MTF.xlsx 분석(7시트 중 5개 데이터·레이아웃 3종·SKU 부재·품절/가능/문의 상태 존재). ADM-CSV-010(정본 적재)과 목적이 달라 별도 화면.
