# UX/UI 업그레이드 스펙 — "확신을 증명하는" 설계

> 목적: 조립PC 구매 고객 전략 분석을 목업 UI·여정에 반영한다.
> 핵심 전환: **말하지 말고 보여준다(Show, don't tell).** 배지·문구 → 실측·실사진·차트·표.
> 시각 자산(사진·아이콘 팩 등)은 **별도 조달**한다. 이 문서는 **슬롯 위치 + 들어갈 내용**만 정의한다.
> 정체성 불변: 결정론 엔진·신뢰 판매자(A-07). 톤: Popcorn Light(D-07).

---

## 0. 설계 원칙 (UI 방향)

### 0.1 확신 3축 → 화면의 책임
파는 것은 스펙이 아니라 **확신**이다. 확신은 3축으로 쪼개지고, 각 축을 담당 화면이 "증명"한다.

| 확신 축 | 고객 질문 | 증명 방식(UI) | 주 담당 화면 |
|---|---|---|---|
| **정확한 상품** | "내 용도에 맞나? 정품·정확한 모델 맞나?" | 용도→결과 매칭(프레임 차트), 부품 풀네임·수량·정품 표기 | S1·S2 |
| **정당한 가격** | "값이 적정한가?" | 개별구매 대비 이득 비교표, 가격 기준시각, 트레이드오프 | S2 |
| **안심 서비스** | "고장·불량 나면? 조립 상태는?" | 검수 체크리스트·24h 번인 리포트·선정리/OS 실사진·A/S 조건표 | S2·S4 |

### 0.2 고객 스펙트럼과 2기어 매핑
고객 유형(게이머/작업/가성비/초보/매니아)은 결국 **감정 축 하나**로 수렴한다: **초보(불안) ↔ 매니아(완성도)**.
- 이미 있는 S1 **2기어**를 이 축의 대리 스위치로 승격한다.
  - `추천 질문 받기`(초보 모드) → **안심·A/S·검수 증거를 앞에**, 스펙 복잡도는 접어둠, 결과를 "쉬운 말"로.
  - `직접 다 입력`(매니아 모드) → **풀 스펙·브랜드·검수 로그·확장성(파워/보드 여유)을 앞에**, 수치·근거 밀도 높임.
- 용도(게임/사무/영상/방송)는 이 감정 축 위에 **얹히는 필터**다(직교).

### 0.3 시각 우선 원칙
- 긴 텍스트 블록 금지. **차트·표·도형·아이콘·실사진**으로 "결과"를 먼저 보여주고 텍스트는 캡션.
- 색 의존 금지(D-05) 유지: 상태는 아이콘+텍스트 병기.
- 정체성 톤(Popcorn Light) 유지: 캔버스 `#F0F5F9`, 카드 `#fff`, 액센트 `#65A1D4`, 헤드라인 Hanken Grotesk.

## 1. 시각 슬롯 규약 ★ (위치만 잡고 내용 정의)

모든 시각 요소는 **슬롯**으로 마크업한다. 자산이 오기 전엔 **내용 라벨이 박힌 회색 placeholder**로 렌더된다. 나중에 자산만 끼우면 된다.

### 1.1 공통 마크업
```html
<figure class="vslot" data-visual="<고유 id>" data-visual-type="image|gallery|chart|table|diagram|icon"
        data-visual-spec="여기에 '무슨 내용/데이터'인지 한 줄 정의">
  <!-- placeholder: 자산 오기 전 표시 -->
  <div class="vslot__ph" aria-hidden="true"><span class="vslot__label">[내용 라벨]</span></div>
  <figcaption class="vslot__cap">캡션(결과 요약)</figcaption>
</figure>
```
- `.vslot__ph` = 회색 박스(점선 테두리 + 라벨 + 타입 아이콘). 자산 넣으면 `hidden`.
- **자산 교체 규약**
  - `image`/`gallery` → `<img data-slot alt="…(정의)">` 삽입 → placeholder 숨김.
  - `chart` → `<canvas data-chart="<종류>">` + 데이터는 `data-visual-spec`에 정의(더미 JS로 렌더).
  - `table`/`diagram` → 실제 HTML로 지금 구현(자산 불필요).
  - `icon` → 단색 인라인 SVG(D-04) 지금 구현.

### 1.2 슬롯 타입 5종
| 타입 | 용도 | 지금 구현? |
|---|---|---|
| `image` | 단일 실사진(예: 선정리 완료샷) | 자산 대기(placeholder) |
| `gallery` | 실사진 3~6장 그리드(검수·조립·후기) | 자산 대기 |
| `chart` | 성능(프레임)·번인 온도·가격구성 | 지금 canvas+더미 데이터 |
| `table` | 비교표·부품 명세·A/S 조건 | 지금 구현 |
| `diagram` | 흐름·타임라인·확장성 도식 | 지금 인라인 SVG |

## 2. 화면별 업그레이드 (4테마 × 슬롯 정의)

테마 표기: ①안심증거 ②톤분기 ③결정촉진 ④가격비교

### 2.1 main-landing (소개)
- **① 검수 증거 프리뷰** — `gallery` 슬롯 `evidence-preview` : "검수 완료 실사진 4장(선정리/번인화면/포장/OS부팅)". 캡션 "모든 PC는 이 과정을 거칩니다."
- **③ 실구매 후기** — `gallery` 슬롯 `review-photos` : "고객 조립 실사진 + 별점 3~6장".
- **① A/S 요약** — `icon`+`table` 미니 : "무상 N년 · 출장/택배 부담 주체 · 초기불량 교환일".
- (기존 히어로 영상·트러스트 배지 유지.)

### 2.2 S0 (입력 게이트)
- **② 유형 힌트(선택)** — 입력창 아래 보조 pill 2종 도식: "처음이라 불안해요 / 스펙 잘 알아요" → S1 진입 기어 프리셋(초보/매니아). `diagram`(작은 토글 도형). *미정: 넣을지(아래 4장).* 
- **③ 라이브 신호 강화** — `icon`+수치: 실재고 카운트(기존) + "오늘 견적 N건".
- (기존 근거 미리보기·spec preview 유지, 실사진 슬롯화.)

### 2.3 S1 (대화 세션) — ②톤분기의 집
- **② 기어별 강조 스위칭**(새 화면 없음):
  - 초보 모드: AI 버블이 **안심 문구 우선**("이 조합은 조립·A/S까지 저희가 책임집니다"), 질문은 쉬운 말, 후보 풀 카운터 옆 **"검수·A/S 포함"** 배지.
  - 매니아 모드: 파싱 결과에 **풀 스펙·브랜드·확장성 힌트**, 후보 풀 옆 **필터 로그**(어떤 하드필터가 몇 개 걸렀는지) `diagram`.
- **③ 후보 풀 카운터**(기존 검증 파이프라인) 유지 + 매니아 모드에서 필터 단계별 수치 노출.

### 2.4 S2 (제안서) — ★ ①③④가 모두 착지하는 핵심 화면
섹션 순서: 구성요약(호환배지 먼저) → **안심 증거** → 근거 → **가격 이득 비교** → 결정촉진 → CTA.

- **① 안심 증거 모듈** (신설, 근거 섹션 위/아래):
  - `table` `inspection-checklist` : **검수 체크리스트** — 초기불량 검사 ✓ / 발열 실측(°C) / 전력 실측(W) / 소음(dB) / 포트·부팅 확인. 각 행 상태+실측치.
  - `chart` `burnin-graph` : **24h 번인테스트** — 시간축 온도/부하 라인 그래프 + "PASS" 배지. 데이터: 0~24h CPU/GPU 온도 더미.
  - `gallery` `build-photos` : **선정리·OS 설치 완료 실사진** 3~4장(내부 선정리/케이블/OS 부팅화면/포장). 자산 대기.
  - `table` `as-terms` : **A/S 조건표** — 무상기간 · 범위 · 출장/택배 부담 · 초기불량 교환 기한 · 연락 채널.
- **③ 결정 촉진**:
  - `icon`+수치 `stock-urgency` : "이 구성 재고 N대 · 당일 출고 마감 HH:MM".
  - `gallery` `review-photos` : 실구매 후기(사진+별점+한줄).
  - 가격 신선도(기존 price_asof) 강조.
- **④ 가격 이득 비교표**:
  - `table` `price-benefit` : **개별 구매 vs 우리** — 행: 부품별 개별가 합계 / 조립비 / 총액 / **이득(절약액·%)**. "직접 사서 조립하면 X원 + 시간·리스크, 우리는 Y원(검수·A/S 포함)."
  - `chart` `price-pie`(기존) : 가격 구성.
- 성능: `chart` `low-fps` (기존, 게임별 1% Low FPS) — ②초보모드에선 "이 게임 상옵 N프레임"으로 캡션 쉬운 말.

### 2.5 S3 (부품 변경)
- **② 매니아 확장성 도식** — `diagram` `expandability` : 파워 여유(W)·보드 슬롯·베이 여유를 게이지 도형으로. "업그레이드 여지" 시각화.
- (기존 안전 대안·연쇄 스왑·호환 재검증 유지.)

### 2.6 S4 (주문 확인)
- **① 최종 안심 증거 요약** — S2의 안심 모듈 축약 재no出: 검수 ✓ · 번인 PASS · A/S 조건 미니 `table`.
- **배송 타임라인** — `diagram` `delivery-timeline` : 결제 → 검수 → 번인 → 포장 → 출고(당일) 5스텝 도형.
- (기존 조립비 투명·재고 재확인·로그인 게이트 유지.)

## 3. 여정 케이스 매트릭스 ★ (입력 · 묻는 데이터 · 결과)

고객이 S0/S1에 남기는 **입력**, 우리가 추가로 **묻는 데이터**, 최종 **결과(강조 톤)**를 유형별로. (기어=감정 톤, 용도=필터)

| # | 유형(기어) | S0 입력 예시 | S1에서 추가로 묻는 것 | 결과·강조 톤 |
|---|---|---|---|---|
| 1 | **초보·게이머** (추천) | "배그 잘 되는 100만원대" | 게임 무게(가벼운/인기/무거운), 예산 확인만 | "배그 상옵 **평균 N프레임**" 프레임 차트 + **안심 증거(검수·A/S) 크게** + 쉬운 말 |
| 2 | **매니아·게이머** (직접) | "9800X3D+4080S, 200만, 3열 수랭, 화이트" | 부족 필수값만(케이스 규격 등) + 확장성 선호 | 풀 스펙·브랜드·**필터 로그**·**확장성 도식**·번인 그래프. 수치 밀도 高 |
| 3 | **작업용(영상편집)** | "프리미어 4K 편집용 150만" | 동시작업 규모(타임라인 트랙/해상도), 저장 용량 | CPU 멀티코어·RAM·NVMe 강조 + "**4K 프리뷰 렉 없음**" 결과 + 안정성(번인) |
| 4 | **가성비형** | "예산 80만, 최대 성능" | 우선순위(게임 vs 범용), 중고/신품 | **가격 이득 비교표 크게** + 병목 없는 조합 근거 + "이 예산 최적 점수" |
| 5 | **사무·초보** | "인터넷·문서용 저렴하게" | 화면 수·용도 확인 | 조용함·저전력·**A/S 안심** + 과사양 경고("이 이상은 낭비") |
| 6 | **입문·불안형** | 칩만 누름(용도만) | 최소 질문(예산 범위 1개) | **불안 해소 전면**: "조립·검수·A/S 다 포함, 실패 없음" + 검수 실사진 + 후기 |

> 공통 규칙: **초보일수록 결과를 "쉬운 말+실사진"으로, 매니아일수록 "수치+로그+확장성"으로.** 같은 결정론 엔진 결과를 **표현만** 기어로 바꾼다(상태·근거는 단일 원천, UX-04 유지).

## 4. 미정 결정 (임의로 정하지 않음 — 확인 필요)
- S0에 "초보/매니아" 명시 선택 pill을 둘지, 아니면 S1 기어로만 자연 분기할지. (권고: S1 기어로만 — 화면 단순)
- 안심 증거(번인 그래프·검수 실측)의 **데이터 출처** — 실제 검수 파이프라인 연동 전까지 더미. 실장 시 관리자 입력/장비 로그 연동 필요.
- A/S 조건표의 **실제 정책 값**(무상기간·부담 주체) — 사업 정책 확정 필요.
- 가격 이득 비교표의 "개별 구매가" 출처 — 입점가/내부 원가 기준(외부 크롤링 아님, A-05).
- 사후관리(세팅 가이드·업글 알림·재구매 할인)는 **현 퍼널 밖 별도 트랙** — 범위·화면 추후.

## 5. 구현 순서 제안
1. **시각 슬롯 규약(§1)을 shared CSS/컨벤션으로 먼저** — placeholder 스타일 1벌.
2. **S2 안심 증거 모듈(①) + 가격 이득 비교표(④)** — 레버 최대, 한 화면에 집약.
3. **S1 기어 톤 분기(②)** — 기존 구조 위 강조 스위칭.
4. **S2 결정 촉진(③)** + main-landing 증거 프리뷰.
5. S3 확장성 도식 · S4 안심 요약·배송 타임라인.
6. 여정 케이스(§3)를 S1 질문 분기·S2 결과 카피에 반영.
