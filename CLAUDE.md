## 지금 단계 = 모델링 + DB·API 구현 (2026-07-23 개정)
- **목업 = 스펙.** UI 목업(HTML/CSS/JS)이 1차 산출물이며, 손으로 검증된 화면 범위가 구현 범위를 정의한다.
- **DB 구축 완료(2026-07-23)**: Cloud SQL `popcorn_pc`에 스키마(테이블 46+뷰 2)·시드 적용됨. 스키마 단일 원천 = `docs/06_db-erd.md` Ver 4.0, 실행본 = `db/`(Alembic). 접속 정보 = `docs/infra/GCP_SETUP.md`(로컬 전용). 스키마 변경은 **ERD 개정 → 새 마이그레이션** 순서.
- **로컬 API 슬라이스 승인(2026-07-23)**: 스택 = FastAPI(`api/`), 방식 = **화면 단위 수직 슬라이스**(DB→API→목업 바인딩 관통, 이음새 검증 후 다음 화면). 첫 슬라이스 = ADM-PRD-010 상품 목록 읽기 1본. 목업은 API 실패 시 더미 폴백(Firebase 검토 배포 무영향). 새 화면 연동은 슬라이스 단위로 계획 승인 후 진행.
- **여전히 금지(별도 승인 필요):** 실제 LLM 연동(보류 결정 2026-07-21: 착수 시 팝콘톡 한정·공개 노출·Opus 4.8) · 운영 배포(서버 구동은 로컬 전용).
- 충돌 요청이 오면 코드 짜지 말고 먼저 물어라. **비밀값(DB 비밀번호·API 키)은 리포·설정·메모리에 절대 저장 금지.**

## 작업 방식 (속도 핵심 — 이게 느림을 잡는다)
1. **탐색 최소화.** 작업 전 전체 코드베이스를 훑지 마라. 필요한 파일만 열어라. 어떤 파일을 볼지 모르면 먼저 물어라.
2. **한 번에 하나.** 큰 작업을 한 요청에 몰지 마라. "S2 전체" 대신 "S2 섹션1"처럼 쪼개라.
3. **계획 먼저, 코드 나중.** 파일을 만들기 전 3~5줄로 접근을 요약하고 승인받아라(Plan Mode 선호).
4. **끝나면 자체 검증.** 목업은 브라우저에서 열리는지·JS 콘솔 에러 없는지 스스로 점검 후 보고.
5. **말은 짧게.** 턴 사이 설명은 최소, 최종 보고는 요점만. 장황한 서술 금지.
6. **세션은 일회용.** 결정은 이 파일/`docs/decisions/decision-log.md`에 기록. 긴 세션에 기억을 의존하지 마라. 컨텍스트가 길어지면 `/clear` 후 재시작.

## 디자인 계약
- 토큰 단일원천 = `design-system/tokens.css` `:root`. 임의 HEX/px 금지.
- 서체: Inter(라틴·숫자) + Pretendard(한글). 숫자는 tabular.
- 색: primary `#005ab3`/hover `#0082fd`, tertiary `#853d97`, error `#b31b25`. **초록 없음** → 통과=primary, 위험=error, 항상 텍스트 병기.
- **아이콘: Streamline Ultimate Color (CC BY 4.0, 출처 표기 필수)** — 2026-07-16 사용자 지시로 단색 정책 폐기. 파일=`mockups/shared/icons/su/`, 매핑=`_map.json`, 런타임 스와퍼=`mockups/shared/su-icons.js`(전 페이지 주입, 출처 표기 자동 삽입). 새 아이콘 필요 시 iconify에서 받아 `ms-`/`ft-` 규칙으로 추가.
- 마크업: `data-screen-id`/`data-domain`/`data-bind`/`data-repeat`/`data-action`, 예외는 `hidden`. 한국어 UI, 768px 세로 스택.

## 상태 관리 원칙 (S1에서 확립)
- 제약은 **객체 하나가 단일 원천**. 칩·자유입력·파싱이 전부 그 하나만 갱신.
- 기어/뷰 전환은 상태를 리셋하지 않는다.
- 후보 풀 카운터 = 완성품 대수 아님. **전체 부품(26,480개)에서 조건 통과 부품 수.**
